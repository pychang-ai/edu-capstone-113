# 培訓 4：論文閱讀方法

## 學習目標

1. 學會快速閱讀一篇學術論文的方法
2. 能用結構化模板整理論文重點
3. 理解論文的組成結構（Abstract → Conclusion → Method → Results）
4. 產出一篇完整的論文摘要筆記

## 為什麼要讀論文？

做研究不是從零開始。別人已經做過的事，你不需要重做。論文告訴你：
- 這個問題目前做到什麼程度（Related Work）
- 別人用什麼方法（Methodology）
- 結果好不好（Results）
- 還有什麼沒做（Future Work）→ 這就是你可以做的

## 論文閱讀三遍法

### 第一遍：5 分鐘掃描（決定要不要讀）

只看：
1. **標題**：這篇在做什麼？
2. **摘要（Abstract）**：用什麼方法？結果如何？
3. **結論（Conclusion）**：作者自己認為最重要的發現是什麼？
4. **圖表**：快速掃過所有圖和表，看得出大概在做什麼

第一遍後你應該能回答：這篇和我的研究有關嗎？

### 第二遍：30 分鐘精讀（理解方法和結果）

看：
1. **Introduction**：問題背景、研究缺口（Gap）、本文貢獻
2. **Methodology**：用了什麼模型/演算法/資料集
3. **Results**：表格和數字，和其他方法比起來如何
4. 跳過數學推導和實作細節

第二遍後你應該能用自己的話說出這篇論文在做什麼。

### 第三遍：深讀（如果和你的研究直接相關）

- 看數學公式和推導
- 看實驗的超參數設定
- 思考能不能重現、能不能改進
- 通常只有核心文獻需要讀到第三遍

## 論文摘要模板

```markdown
# 論文摘要：（論文標題）

## 書目資訊

| 項目 | 內容 |
|------|------|
| 標題 | |
| 作者 | |
| 期刊/研討會 | |
| 年份 | |
| DOI/連結 | |

## 一句話摘要

（用一句話說明這篇論文在做什麼，30 字以內）

## 研究問題

（作者想解決什麼問題？為什麼這個問題重要？）

## 方法

（用了什麼方法？資料集是什麼？模型架構是什麼？）

## 主要結果

（最重要的數字或發現是什麼？和其他方法比起來如何？）

## 和我的研究的關聯

（這篇對我的專題有什麼啟發？能用到什麼？）

## 我的疑問

（讀完後還不理解的地方，或覺得可以改進的地方）
```

## 練習作業

老師會指定一篇論文給你閱讀。完成以下任務，將結果放在 `members/你的學號/paper-summary.md`：

1. 用上面的模板，寫出完整的論文摘要
2. 每個欄位都要填（「我的疑問」至少寫 1 點）
3. 「一句話摘要」要用你自己的話，不能直接翻譯 Abstract
